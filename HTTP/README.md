# HTTP狀態碼：
在上一個任務中，您了解到當HTTP服務器響應時，第一行總是包含一個狀態代碼，通知客戶端其請求的結果以及可能如何處理它。這些狀態代碼可以分為 5 個不同的範圍：

```
100-199 - 信息響應	發送這些信息是為了告訴客戶他們的請求的第一部分已被接受，他們應該繼續發送其餘的請求。這些代碼不再很常見。

200-299 - 成功	這個範圍的狀態碼用於告訴客戶端他們的請求是成功的。

300-399 - 重定向	這些用於將客戶端的請求重定向到另一個資源。這可以是不同的網頁或完全不同的網站。

400-499 - 客戶端錯誤	用於通知客戶他們的請求有錯誤。
500-599 - 服務器錯誤	這是為服務器端發生的錯誤保留的，通常表明服務器處理請求存在相當大的問題。
```
# 常見的HTTP狀態碼：

有很多不同的 HTTP 狀態代碼，這還不包括應用程序甚至可以定義自己的事實，我們將介紹您可能遇到的最常見的 HTTP 響應：

```
200 - 好的	請求已成功完成。

201 - 已創建	已創建資源（例如新用戶或新博客文章）。

301 - 永久重定向	這會將客戶端的瀏覽器重定向到一個新網頁，或者告訴搜索引擎該頁面已移動到其他地方並改為查看那裡。

302 - 臨時重定向	與上述永久重定向類似，但顧名思義，這只是臨時更改，可能在不久的將來再次更改。

400 - 錯誤請求	這告訴瀏覽器他們的請求中有問題或丟失。如果正在請求的 Web 服務器資源需要客戶端未發送的某個參數，則有時可以使用此方法。

401 - 未授權	在您獲得 Web 應用程序授權之前，您目前無法查看此資源，最常見的是使用用戶名和密碼。

403 - 禁止	無論您是否登錄，您都無權查看此資源。

405 - 不允許的方法	該資源不允許此方法請求，例如，您向資源 /create-account 發送了一個 GET 請求，而它原本期望一個 POST 請求。

404頁面不存在	您請求的頁面/資源不存在。

500 - 內部服務錯誤	服務器在您的請求中遇到了某種錯誤，它不知道如何正確處理。

503服務不可用	
此服務器無法處理您的請求，因為它已超載或停機維護。
```
